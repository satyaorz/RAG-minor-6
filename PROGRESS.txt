SUMMARY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
TreeQA is a hallucination-aware multi-hop RAG system. It decomposes complex
questions into a reasoning tree, retrieves evidence for each node via hybrid
vector + graph lookup, generates LLM answers, and validates each answer for
hallucinations before producing a final synthesised response.

The project is a fully working local web application with a FastAPI backend
and a single-page frontend. All core components are built and tested.

[x] FastAPI backend (src/treeqa/api/app.py)
      GET  /             → serves the SPA (index.html)
      GET  /api/health   → {"status":"ok"}
      GET  /api/documents → list uploaded documents with name/size
      POST /api/run      → run full TreeQA pipeline, return reasoning tree
      POST /api/upload   → upload .txt/.md, re-index, reload pipeline
      POST /api/compare  → retrieval-only baseline vs TreeQA side-by-side

[x] Single-page frontend (src/treeqa/ui/index.html)
      — Query box with Ctrl+Enter shortcut
      — AbortController with 2-minute client timeout + elapsed counter
      — Cancel button during long requests
      — Document upload panel (drag-and-drop, shows file list with sizes)
      — Compare mode toggle (Traditional RAG vs TreeQA)
      — 3-tab layout: 🌳 TreeQA | ⚖ Compare | 📊 Charts
      — TreeQA tab: stats bar, final answer card, expandable reasoning tree
        with per-node confidence bars, evidence toggles, validation badges
      — Compare tab: side-by-side columns, confidence chips, doc counts
      — Charts tab: Chart.js visualisations
          · Method comparison bar chart (confidence, docs, hops)
          · Per-node confidence horizontal bars (colour-coded by status)
          · Top-12 retrieval scores by source type (vector / graph)

[x] Traditional RAG baseline (compare mode)
      — Pure retrieval, no LLM calls — returns top-3 doc snippets in <1s
      — Represents naive keyword-search RAG for meaningful comparison
      — No timeout risk; only TreeQA's LLM calls count against the limit

[x] Async pipeline execution
      — asyncio.wait_for with 110s timeout wraps blocking pipeline.run()
      — ThreadPoolExecutor(max_workers=4) prevents blocking the event loop
      — HTTP 504 returned on timeout with user-friendly message

[x] python-multipart added to [api] optional deps in pyproject.toml

  pip install -e ".[api]"
  cp .env.example .env          # add OPENROUTER_API_KEY
  PYTHONPATH=src uvicorn treeqa.api.app:app --reload --port 8000
  # open http://localhost:8000

  Run tests (no API key needed):
  PYTHONPATH=src pytest tests/ -q
